# Fontainor Developer Handoff Standards

This profile acts as the core memory state anchor for all AI engineering collaborators interacting with the Fontainor Protocol workspace.

## 🚨 Rigid Development Constraints & Rules

1. **Surgical Search and Replace Models ONLY:** Always output small, targeted `sed` replacement blocks, never large, complete file overwrites.
2. **Zero-Overhead Serverless Ideology:** All features must maintain the serverless, decentralized architecture.
3. **Always Ask for Code Context:** Use tools like `sed` and `grep` to verify file context before changes.
4. **Proxy Path Integrity Checks:** Ensure correct, slash-free formatting for network gateway addresses.

## 🛠 Active Workstation Command Directory
*   `sed -n 's,p' file` for editing.
*   `grep -rn "pattern" src/` for searching.
*   `rm -f` to clean local `arlocal.db` and registry.
