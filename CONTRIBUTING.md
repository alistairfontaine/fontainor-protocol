# Fontainor v0.2 Development Guidelines

1. **Protocol Sovereignty:** Every feature must respect the "Asset-First" philosophy. No streaming metrics, no vanity UI.
2. **Persistence:** All data structures must be compatible with Arweave/Permaweb storage.
3. **Collector's Equity:** The core logic of resale royalties must remain the priority in the smart contract layer.
4. **Lean Stack:** Keep the protocol architecture lean. If a feature doesn't directly contribute to the minting or resale of Equity Editions, it is out of scope.
